# Producto y loop de juego

## Requisitos

- `REQ-PRODUCT-001`: simulación política seria, single-player, en un Chile contemporáneo ficticio.
- El jugador toma decisiones estratégicas; no existen minijuegos como núcleo.
- El loop es `briefing → decisiones → avance temporal → reporte causal`.
- El tiempo avanza en 1, 4 o 12 semanas y puede detenerse ante decisiones bloqueantes.
- Las métricas visibles, tendencias y causas deben permitir comprender las consecuencias.
- El mapa regional y los paneles consumen el runtime; no contienen reglas de simulación.

## Alcance inicial

Primero se construye un vertical slice headless. UI final, arte, catálogo masivo, metaprogresión profunda y ruta constitucional completa permanecen diferidos.
