# Contratos activos

Solo los documentos de esta carpeta con `authority: canonical` son normativos. Cada `CON-*` debe ser único.

Los contratos fijan comportamiento e invariantes, no el estado de implementación. Una regla documentada aquí puede estar todavía pendiente; consultar siempre [`../implementation/STATUS.md`](../implementation/STATUS.md).

## Contratos vigentes

- [`CON-PRODUCT-001`](product.md): visión y loop del producto.
- [`CON-SIM-001`](simulation-core.md): estado, targets, efectos y tiempo.
- [`CON-TRACE-001`](causality.md): causalidad y reportes.
- [`CON-CONTENT-001`](content.md): Content Pack y validación.
- [`CON-PERSIST-001`](persistence.md): save/load y migraciones.
- [`CON-SYSTEMS-001`](game-systems.md): eventos, legislación y territorio.
- [`CON-DOCS-001`](documentation.md): contexto y trazabilidad del proyecto.
