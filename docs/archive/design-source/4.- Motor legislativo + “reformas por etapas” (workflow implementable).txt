2.4 Progreso de etapa (WORK vs VOTE)

Cada etapa tiene:

requiredS = ReformTemplate.base_difficulty (default 10000)

stage_weightS (por defecto):

WORK: 100*S

VOTE: 120*S (votar es más duro)

(ajustable por template, pero el motor ya lo soporta)

WORK (comisión/negociación): cada tick:

supportS = el de la cámara indicada por stage.chamber (si BOTH usa el mínimo de ambas).

throughputS = 200*S * (metrics.governabilityS / (100*S)) * (supportS / (100*S))

progress += throughputS

Si progress >= requiredS * stage_weight ⇒ pasa a la siguiente etapa.

VOTE (votación):

Se acumula “whip work” igual que WORK pero con threshold:

throughputS *= clamp((supportS - 45*S) / (55*S), 0..1) (si estás lejos, casi no avanzas)

Cuando progress >= requiredS * stage_weight, se ejecuta un check determinista:

si supportS >= pass_thresholdS ⇒ etapa completada

si no ⇒ attempts_in_stage += 1, progress = 6000 (retrocede a 60%) y aplica coste sistémico:

metrics.internal_cohesion -= 1*S

metrics.public_agenda += 1*S

(y el motor puede encolar un evento plantilla de “derrota legislativa” si existe; si no, solo estos deltas)

Esto implementa “estancarse/retroceder por crisis/presión/fractura” sin RNG, y mantiene legibilidad para el reporte causal
